# Security Policy

## Reporting a vulnerability

If you find a security issue, please do not open a public issue with sensitive details.

Contact the maintainer privately through GitHub or LinkedIn and include:

- a short description of the issue;
- steps to reproduce it;
- affected files or versions;
- whether any personal data could be exposed.

## Sensitive data policy

Do not commit real documents, generated mappings, audit logs, restored files, or screenshots containing personal data.

The repository is intended to contain only:

- source code;
- documentation;
- tests;
- synthetic examples.

## Local-first assumptions

This project is designed to process documents locally. The mapping files generated by the app may contain original personal data and should remain on the user's machine.

## Disclaimer

This project is not a legal anonymization guarantee and does not provide compliance certification. Human review remains required before sharing any output.
